# Sets

A set is an ordered list of patches. Sets are useful for organizing a live performance sequence or a workflow that steps through related configurations in a defined order.

## Creating a set

In **Sets** mode, click **+** in the set list (or Cmd+N) to create a new set.

## Adding and reordering patches

When a set is selected, the patches it contains are shown below the set list. Click **Add Patch** at the bottom of that list to add patches from the document. A patch can appear in multiple sets.

Drag a patch row to reorder it within the set. To remove a patch from the set, right-click it and select **Remove from Set** — the patch itself is not deleted.

## Navigating a set

Use the **Left / Right arrow keys** (or Cmd+← / Cmd+→) to step through patches, or click the navigation buttons in the bar at the bottom of the canvas. The current position is shown as "N / Total — Patch Name".

Clicking a patch row in the list also jumps directly to it.

## Read-only canvas

The canvas in Sets mode is read-only. Cables cannot be created or deleted, controls cannot be adjusted, and no context menus appear. The patch notes overlay is still visible and can be clicked to read the notes for the current patch.

To make changes, switch to **Patches** mode, edit the patch there, and return to Sets mode.
